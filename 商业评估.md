# 猫省省 — 商业评估

> 评估日期：2026年6月26日
> 方法论：单位经济学 × 竞争壁垒 × 天花板测算
> 原则：用数字说话，不给软解释

---

## 一、单位经济学：拆到每一块钱

### 收入线硬拆

| 收入线 | 单用户年价值 | 乐观假设 | 残酷假设 |
|--------|------------|---------|---------|
| 会员订阅 68元/年 | 68元 × 付费率 | 付费率 5%，3.4元/用户 | 付费率 1.5%，1.02元/用户 |
| 药品CPS | 年购药3次 × 均佣5元 = 15元 × 使用率 | 使用率 30%，4.5元/用户 | 使用率 10%，1.5元/用户 |
| 保险CPS | 100元 × 购买率 | 购买率 5%，5元/用户 | 购买率 1%，1元/用户 |
| 单次分诊 3.9元 | 3.9 × 年均使用次 × 付费率 | 年均3次 × 非会员10% = 1.17元 | 年均1次 × 5% = 0.20元 |
| **年ARPU** | | **14.07元** | **3.72元** |

### 获客成本

| 渠道 | 单次下载成本 | 能否规模化 |
|------|------------|-----------|
| 豆瓣猫组（内容帖） | ~0元（时间成本） | 否，58万成员但发帖频率受限 |
| 小红书 KOC | 1-5元/曝光 → 下载转化 1-3% → **30-150元/下载** | 可，但贵 |
| 微信猫友群 | ~0元 | 否，量小且需要混群 |
| 应用商店 ASO | 3-8元/下载 | 可，但宠物类搜索量有限 |
| 信息流广告 | 10-30元/下载 | 可，但 ROAS 基本为负 |
| **综合 CAC** | **15-50元/下载** | |

### 单用户回本

```
乐观：ARPU 14.07元，CAC 15元 → 回本周期 = 1.07年
残酷：ARPU 3.72元，CAC 50元 → 永远回不了本
```

**关键发现**：只要 CAC > 15元，且付费率 < 3%，这个模型在数学上就不成立。而中国宠物类 App 的 CAC 几乎不可能压到 15元以下。

## 二、天花板测算：最好的情况能做到多大

### 用户规模

| 层级 | 人数 | 依据 |
|------|------|------|
| 中国猫主人 | ~6,000万 | 2025行业数据 |
| 会下载宠物类App的 | ~1,000万（~17%） | 参考宠智灵600万用户为品类天花板 |
| 我们的可达用户（3年） | 30-80万 | 无资本助推的独立App天花板 |

### 收入天花板

| 用户量 | 年收入（乐观ARPU） | 年收入（保守ARPU） |
|--------|------------------|------------------|
| 10万 | 140万 | 37万 |
| 30万 | 422万 | 112万 |
| 80万 | 1,126万 | 298万 |
| 100万 | 1,407万 | 372万 |

### 扣除成本

| 用户量 | 获客总成本 | 年运营成本（服务器+API+其他） | 年净利（保守） |
|--------|----------|---------------------------|-------------|
| 10万 | 150-500万 | 5万 | 负 |
| 30万 | 450-1500万 | 15万 | 负 |
| 80万 | 1200-4000万 | 30万 | 负 |
| 100万 | 1500-5000万 | 50万 | 负 |

**结论**：以当前定价，无论做到多大规模，获客成本永远大于单用户收入。这是**结构性亏损**，不是执行力问题。

---

## 三、为什么是结构性亏损

三角死局：

```
           低价订阅（68元/年）
                ↓
         LTV 过低（14-176元）
                ↓
    CAC 必须 < 6-59元才能盈利
                ↓
    但宠物类 CAC 不可能低于 15元
                ↓
        用户必须用 2-4 年才能回本
                ↓
    但猫年均生病 2.11 次 → 年打开 < 5 次
                ↓
    低频产品 → 次年留存率极低（<30%）
                ↓
           LTV 进一步压缩
                ↓
           回到起点：死局
```

**根本矛盾**：客单价（68元/年）无法覆盖获客成本（15-50元），且低频使用导致留存天然低。

## 五、竞争响应：如果验证了需求，会发生什么

| 玩家 | 跟进速度 | 跟进方式 | 我们的防御 |
|------|---------|---------|----------|
| 京东健康宠物 | 3个月 | App内加AI分诊入口 | 无，他们有用户、供应链、零获客成本 |

## 六、盈利模型的三种出路

### 路径 A：独立 App 订阅（当前方案）

```
❌ 数学不成立。CAC > LTV，永远烧钱。
```

### 路径 B：微信小程序 + 内容变现

```
小程序获客成本极低（微信生态内传播）
收入不靠订阅，靠：
  ├── 广告（小程序广告组件）
  ├── CPS带货（京东/拼多多 小程序内直接跳转）
  ├── 保险导流（小程序内完成咨询+投保）
  └── 内容付费（平替清单付费解锁）

✅ 获客成本接近 0
✅ 留存靠微信服务通知
✅ 不需要用户"下载App"这个重决策
⚠️ 但 ARPU 比 App 更低（微信生态消费意愿弱）
```

### 路径 C：内容 IP + 带货

```
不做App/小程序，做：
  ├── 小红书账号（平替清单 + AI分诊 demo）
  ├── 抖音账号（猫癣识别 → 平替 → 小黄车带货）
  └── 豆瓣猫组 KOL（持续输出避坑内容）

变现：
  ├── 品牌合作（猫粮/猫砂品牌投放）
  ├── CPS带货（京东联盟）
  └── 知识付费（完整避坑指南）

✅ 零研发成本
✅ 走通后月入 1-5 万（头部宠物博主水平）
❌ 天花板低（年入百万封顶，不是VC故事）
```

## 七、结论

| 维度 | 评分 | 说明 |
|------|------|------|
| 痛点真实性 | 🟢 9/10 | 62%猫奴觉得看病贵，27%怀疑过度医疗 |
| 市场规模 | 🟡 6/10 | 市场大，但可触达的付费用户有限 |
| 商业模式 | 🔴 2/10 | 当前App订阅模型数学不成立 |
| 竞争壁垒 | 🔴 2/10 | 巨头6个月可跟进，内容无壁垒 |
| 监管风险 | 🟡 5/10 | AI分诊分级输出可规避，但不是零风险 |
| **综合** | 🔴 **不推荐独立App** | **在当前定价和获客成本下，商业模式不成立** |

## 八、如果非要做，正确的路径

```
放弃独立App → 改成微信小程序
放弃订阅 → 改成免费 + 广告/CPS/保险导流
放弃"做平台" → 先做内容IP（小红书/抖音）验证需求
```

**第一步**：把平替清单做成小红书/抖音内容，看传播数据。如果单条爆款 10万+ 阅读 → 说明需求存在 → 再考虑产品化。

**第二步**：如果内容验证通过，做微信小程序（不是App）。用户在小程序里：
- 免费看平替清单
- 免费AI分诊（限制次数）→ 付费解锁无限次
- 保险对比 → 一键投保（CPS）

**第三步**：如果小程序月活 > 10万，再考虑独立App。

---

> **一句话结论**：猫省省的商业逻辑作为独立App不成立。但作为内容IP + 微信小程序，是一个值得做的副业项目。当前最该做的是：把平替清单发到小红书，看有没有人看。
