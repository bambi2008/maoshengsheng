# 猫省省 — 产品策划书 v1.0

> 定稿日期：2026年6月26日
>
> 核心定位：**猫奴的工具，不是猫奴的社区。**
> 打开拍照→10秒拿结果→走人。社区是功能的延伸，不是产品的主体。

---

## 一、产品定义

### 一句话

> **猫省省：猫出问题时，拍张照就知道怎么办。**

### 用户画像

| 维度 | 描述 |
|------|------|
| 谁 | 25-35岁女性，一二线城市，养猫 0-3 年 |
| 场景 | 猫吐了/皮肤秃了/拉了软便 → 不确定严不严重 → 想立刻知道要不要去医院 |
| 现有行为 | 打开小红书/豆瓣搜 → 越搜越慌 → 冲医院或网购一堆药 |
| 替代方案 | 百度（吓死人）、小红书（看30分钟帖子）、豆瓣猫组（搜索烂） |
| 付费意愿 | 愿意为"立刻告诉我怎么办"付费，价格锚点是挂号费（30-300元） |

### 不是

| 不是 | 原因 |
|------|------|
| ❌ AI诊断工具 | 误诊法律风险致命 |
| ❌ 社区App | 豆瓣猫组58万人搬不动 |
| ❌ 电商平台 | 京东/淘宝碾压 |
| ❌ 养猫百科 | 小红书的内容厚度无法竞争 |

## 二、产品架构

```
猫省省 App
│
├── 📸 拍照分析（首屏，占80%视觉面积）
│   ├── 拍皮肤（猫癣 · 掉毛 · 黑下巴 · 过敏）
│   ├── 拍呕吐物（毛球 · 食物 · 胃液 · 胆汁 · 血）
│   ├── 拍排泄物（正常 · 软便 · 水便 · 带血 · 便秘）
│   ├── 拍药品/处方（处方OCR → 全网比价）
│   └── 拍商品（猫粮/保健品成分表分析）
│
├── 💊 比价买药
│   ├── 输入药名 → 京东/淘宝/拼多多实时比价
│   ├── 国产替代药推荐（成分相同，价格 1/3）
│   └── 一键跳转购买（CPS佣金）
│
├── 🧠 避坑百科
│   ├── 平替清单（十品类 · 贵价 vs 平替对比）
│   ├── 成分分析（拍照配料表 → 自动评级）
│   └── "这值不值"查询
│
├── 📋 健康档案
│   ├── 驱虫/疫苗提醒
│   ├── 就诊记录（拍照病历自动归档）
│   └── 症状时间线（"上个月也吐过"→自动关联）
│
├── 🛡️ 保险助手
│   ├── 方案对比（众安 vs 平安 vs 太保）
│   ├── 拍照理赔（病历拍照 → 自动填单）
│   └── 续费提醒
│
└── 💬 猫友经验（社区层）
    ├── 不设独立Tab，藏在分析结果页底部
    ├── "37个类似案例的猫奴经验"
    └── 按症状自动聚合，不是时间流
```

## 三、核心功能详解

### 3.1 拍照分析（核心功能，必须做到极致）

**用户路径**：打开App → 相机已就绪 → 拍照 → 等待2-3秒 → 看到结果

```
拍照后AI做的事：

1. 图像识别：这是什么（呕吐物/皮肤/排泄物/药品）
2. 特征提取：颜色/质地/范围/严重程度
3. 相似案例匹配：数据库中相似的案例
4. 输出分层结果：

   🟢 低风险
   "你家猫的皮肤症状和数据库中 89% 的轻度猫癣案例相似。
    常见处理：外涂盐酸特比萘芬乳膏（人药店 15 元），每天 2 次，持续 2 周。
    什么情况需要就医：范围扩大/出现化脓/2周无好转。
    猫友经验：37 个成功治愈的案例 → [展开]"

   🟡 中风险
   "呕吐物含黄色胆汁，可能原因包括：空腹过久/轻微胃炎/毛球刺激。
    建议：观察 6 小时，禁食不禁水，后续喂小份易消化食物。
    如果出现以下情况请就医：反复呕吐/精神萎靡/拒绝饮水。
    猫友经验：15 个类似情况 → [展开]"

   🔴 高风险
   "呕吐物含血丝/咖啡渣样物质，这可能是胃出血的信号。
    建议尽快就医（非急诊，今天内）。
    附近 24 小时宠物医院：[列表]
    就医时带什么：拍照记录 + 呕吐物样本（如有）。
    猫友经验：8 个治疗案例 → [展开]"
```

**关键设计原则**：
- 不说"你的猫得了X病"（诊断），说"和数据库中X案例相似"（信息匹配）
- 不分轻重一律"建议就医"= 产品自杀。轻症要给方案，重症才去医
- 每个结果底部附"猫友经验"——这是AI结果的信任背书。用户信同类案例比信AI多

### 3.2 比价买药

**入口**：拍照分析结果页 → "如果确诊X，常用药比价如下"；也可主Tab直接搜

**体验**：
```
搜索"大宠爱成猫"
  →
  京东：¥68.00（自营）   [去购买]
  淘宝：¥62.50（XX大药房） [去购买]
  拼多多：¥59.90（百亿补贴） [去购买]
  
  成分相同替代药：
  雷米高塞拉菌素：¥28.00 [去购买]
```

**CPS**：京东联盟、淘宝客、拼多多多多客。佣金2-5%。用户省了钱，你赚了佣金，没有利益冲突——因为推荐的是**同成分更便宜的**，不是贵价高佣的。

### 3.3 避坑百科

**两种输入方式**：
- 首页搜"猫窝"/"化毛膏"/"鱼油"
- 拍照商品包装/配料表

**输出**：
```
"K9 宠物鱼油"
成分评级：⭐⭐（Omega-3含量低，添加剂多）
智商税指数：🔴 高
同等效果平替：人用 Now Foods 鱼油（¥68/瓶）
成分对比：[并排展示]
猫友评价：12条 → "猫根本不吃""买了后悔"
```

### 3.4 社区层（不是独立功能，是分析结果的延伸）

**不存在独立社区Tab。** 社区内容只出现在两个地方：

| 位置 | 展示内容 |
|------|---------|
| 分析结果页底部 | "37个类似案例的猫奴经验" |
| 避坑百科详情页 | 该产品的猫友真实评价 |

**社区内容来源**：
- 冷启动：从豆瓣猫组手动精选 200 条高质量帖子（公开信息整理，标注来源）
- 自增长：用户分析结果后可以"补充我的经验"，格式固定（症状→诊断→用药→花费→耗时→建议），不是自由发帖
- 质量筛选：每条经验必须绑定一次真实的分析记录才能发，杜绝水帖和营销号

**为什么这样设计社区**：
- 豆瓣用户不需要另一个"刷帖子的地方"
- 但他们在焦虑时需要看到"有人和我一样"
- 社区内容按**症状聚类**，不是按时间排序——时新性不如相关性重要

## 四、商业模式（修正版）

### 收入线

| 收入线 | 方式 | 单人年价值 | 前提 |
|--------|------|-----------|------|
| 会员订阅 | 无限次分析+比价+避坑+健康档案+保险助手 | 68元/年 | 不靠这个活 |
| 单次分析 | 非会员每次3.9元 | 偶发 | 每天限1次免费 |
| 药品CPS | 比价后跳转京东/淘宝/拼多多 | 3-10元/年 | 透明分佣 |
| 保险CPS | 对比后投保 | 80-160元/单 | 需要保险牌照吗？不需要——导流到众安/平安平台完成投保 |
| 广告 | App内不接品牌广告 | 0元 | 永不接。接广告=背叛 |
| **远期权衡** | **自营OTC药品** | **高毛利** | **需《兽药经营许可证》，MVP不做** |

### 为什么68元能活

不需要花钱买量。获客靠三样：

1. **功能自带传播**：拍照→结果→"分享给猫友群"按钮 → 猫友群看到 → 下载
2. **小红书内容导流**：平替清单/猫癣对比案例 → 留小程序/App入口
3. **豆瓣猫组口碑**：有人发帖问"我家猫这样怎么办" → 评论区有人回"下个猫省省拍照看看"

这三条渠道的获客成本 ≈ 0。不需要信息流投放。

### 财务模型（保守）

| 用户量 | 付费率 | 会员收入 | CPS+保险 | 年总收入 | 成本 | 净利 |
|--------|--------|---------|----------|---------|------|------|
| 1万 | 4% | 2.7万 | 1.5万 | 4.2万 | 3万 | 1.2万 |
| 5万 | 4% | 13.6万 | 7.5万 | 21.1万 | 5万 | 16.1万 |
| 10万 | 4% | 27.2万 | 15万 | 42.2万 | 8万 | 34.2万 |
| 30万 | 4% | 81.6万 | 45万 | 126.6万 | 20万 | 106.6万 |
| 100万 | 4% | 272万 | 150万 | 422万 | 50万 | 372万 |

成本构成：服务器 + API + 人工维护。

**关键假设**：不做付费投放。如果获客成本为零，4% 付费率足以养活一个人。100 万用户需要 3-5 年自然增长，不存在短期暴富。

## 五、技术架构

### 5.1 技术栈

| 层 | 选型 | 理由 |
|----|------|------|
| 前端 | Flutter | 已有 SnapDeduct 经验；一套代码 iOS+Android |
| 后端 | Python FastAPI | 轻量、AI 生态好、部署简单 |
| 数据库 | PostgreSQL + pgvector | 关系型存用户数据，vector存图像嵌入用于相似案例搜索 |
| 缓存 | Redis | 热门比价数据缓存 |
| 存储 | 阿里云 OSS / Cloudflare R2 | 用户上传图片 |
| AI推理 | DeepSeek V4 Pro API (+ MiniMax 备选) | 多模态识别 |
| 图像嵌入 | CLIP / 本地轻量模型 | 相似案例匹配 |
| OCR | 百度OCR / 腾讯OCR | 处方、配料表识别 |
| 比价数据 | 京东联盟API + 淘宝客API + 爬虫补全 | 药品价格 |
| 部署 | 阿里云 ECS（国内）+ Cloudflare（全球） | 国内用户访问速度优先 |

### 5.2 AI 流水线

```
用户拍照
    │
    ▼
┌─────────────┐
│ 1. 图像分类   │ ← 轻量模型本地/边缘（判断：皮肤/呕吐/排泄/药品）
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 2. 特征提取   │ ← CLIP 生成图像 embedding
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 3. 案例匹配   │ ← pgvector 搜索相似案例（已有诊断结果的历史案例）
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 4. LLM 分析  │ ← DeepSeek V4 Pro：输入=图像描述+相似案例摘要+猫基本信息
│              │    输出=分层建议（风险等级+可能原因+处理方案+就医指征）
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 5. 结果渲染   │ ← 前端展示：风险等级 + 建议 + 比价链接 + 猫友案例
└─────────────┘
```

**关键**：LLM 不做诊断，只做信息综合。输入中必须包含"相似案例"作为参考锚点——让 LLM 基于真实案例来输出建议，而不是凭空判断。

### 5.3 数据流

```
全量数据存 PostgreSQL
  ├── users：用户 + 会员状态
  ├── pets：猫的基本信息（品种/年龄/体重/病史）
  ├── analyses：每次分析记录（图片URL + 结果 + 用户反馈）
  ├── cases：精选案例库（症状描述+诊断+用药+花费+时长）← 社区层数据基础
  ├── products：药品/保健品/用品数据库
  └── prices：比价数据（定时更新）

相似案例匹配：cases 表的 embedding 存在 pgvector，分析时做向量搜索
```

## 六、冷启动策略

### 第一阶段：内容蓄水（第 1-4 周，不写代码）

- 从豆瓣猫组手动精选 200 条高质量真实案例，整理为标准格式（症状→诊断→用药→花费→建议）
- 按症状分类入库，作为 AI 的初始参考库
- 同步将平替清单等做成小红书/抖音内容，测试传播

### 第二阶段：MVP 开发（第 5-8 周）

- Flutter App：拍照+结果页+比价（三个Tab，其他先不做）
- 后端：FastAPI + PostgreSQL + pgvector + DeepSeek API
- 初始案例库 200 条导入
- 仅支持 3 种拍照类型：皮肤/呕吐物/排泄物

### 第三阶段：种子用户（第 9-12 周）

- 豆瓣猫组发帖："做了个拍照识别猫癣的工具，求测试" → 完全产品导向，不发软文
- 首批 100 个用户 → 收集反馈 → 迭代
- 用户每完成一次有意义的分析 → 鼓励分享结果到猫友群

### 第四阶段：功能扩展（第 13 周+）

- 加入比价模块
- 加入避坑百科
- 加入健康档案
- 加入保险助手
- 案例库从手动精选 → 用户贡献（审核制）

## 七、竞争对比（澄清版）

| 竞品 | 他们是做什么的 | 我们不是他们的对手 | 他们替代不了我们 |
|------|-------------|-----------------|----------------|
| 豆瓣猫组 | 社区讨论（内容厚度） | 我们不做社区 | 豆瓣做不到拍照→即时分析 |
| 小红书 | 内容种草（传播力） | 我们不做内容平台 | 小红书做不到结构化分析 |
| 宠智灵 | B2B兽医工具（B端赚钱） | 我们不做B端 | 宠智灵C端免费但不敢真心帮用户省钱 |
| 京东健康宠物 | 卖药（供应链） | 我们不做自营 | 京东不做AI分析 |
| 蘑菇宠医 | 在线兽医（人力） | 我们不做问诊 | 蘑菇不做自动化+比价 |

## 八、关键风险

| 风险 | 等级 | 应对 |
|------|------|------|
| AI分析不准确 | 🔴 高 | 1)不做诊断只做信息匹配 2)每结果附相似案例 3)轻症给方案/重症推就医 4)显眼位置免责 |
| 冷启动案例库不够 | 🟡 中 | 手动精选200条豆瓣帖子（公开信息整理） |
| 竞品跟进 | 🟡 中 | 案例库是时间壁垒；用户体验是执行壁垒 |
| 用户不付费 | 🟡 中 | 4%付费率假设需验证；保险CPS弹性更大 |
| 比价数据维护 | 🟢 低 | 初期只做Top50药品，手动更新；后期接API |
